SkillShare Project Overview
===========================

What SkillShare is
------------------
SkillShare is a Flutter-based, multi-role marketplace app for skilled professionals to showcase their work, publish portfolios, sell products, receive service requests, apply for jobs, chat with customers and companies, and be managed by admins.

The project is backed by Firebase and is designed around four main user roles:
- Customer
- Skilled Person
- Company
- Admin

High-Level App Flow
-------------------
1. App starts in lib/main.dart.
2. Firebase is initialized.
3. Firestore settings are configured differently for web and mobile.
4. The app loads provider state for auth, user data, and app lock.
5. InternetRequiredGate and AppLockGate wrap the app shell.
6. SplashScreen checks FirebaseAuth currentUser.
7. If a valid session exists, the app routes to MainNavigation.
8. If not, the app routes to LoginScreen.
9. After login or signup, the app again routes into MainNavigation.
10. MainNavigation and MainScreen act as the main shell for feature tabs.

Primary Navigation / User Journey
----------------------------------
The core user journey is:
Splash -> Login/Signup -> MainNavigation -> Feature Tabs

The main feature areas are:
- Home / Explore
- Shop
- Jobs
- Chat
- Profile

Role-specific flows branch from the profile/setup and admin areas:
- Skilled users can build a profile, portfolio, and shop.
- Customers can browse experts, message them, and buy products.
- Companies can post jobs, hire professionals, and chat for hiring.
- Admins can moderate users, products, verification, and platform data.

Project Structure Notes
-----------------------
- There are 103 Dart files under lib in this workspace.
- The app is organized by feature folders rather than a single flat screen folder.
- Main user flows are role-aware, with different screens and actions depending on whether the user is a customer, skilled person, company, or admin.
- Firestore is the central source of truth for users, profiles, products, jobs, chats, orders, reviews, appeals, and service requests.

Firestore / Data Flow Summary
-----------------------------
- AuthService creates or loads the core user document.
- UserProvider and AuthProvider keep user state available to the UI.
- FirestoreService handles the majority of CRUD, stream, and business logic.
- ChatService manages messaging and chat thread behavior.
- PortfolioService and CloudinaryService support media-heavy profile and portfolio flows.
- Admin and delivery services extend the same Firebase-backed data model for moderation and fulfillment.

Short Version
-------------
SkillShare is a role-based Flutter marketplace and professional network with Firebase backend, built to connect skilled people, customers, companies, and admins through profiles, portfolios, products, jobs, chat, and delivery workflows.
